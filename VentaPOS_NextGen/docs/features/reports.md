# تقارير النظام (Reports)

يوفر نظام VentaPOS مجموعة شاملة من ٨ تقارير تفصيلية تغطي جميع جوانب العمل التجاري.

---

## ملخص التقارير

| # | التقرير | الغرض | مسار الـ API |
| :--- | :--- | :--- | :--- |
| 1 | **ملخص لوحة القيادة (Dashboard)** | عرض المؤشرات الرئيسية للأداء (KPIs) مثل المبيعات، تكلفة البضاعة، إجمالي المصروفات والمكسب الصافي، بالإضافة إلى تفاصيل حركة السيولة النقدية. | `/reports/dashboard/` |
| 2 | **درج الخزينة (Cash Drawer)** | تفصيل حركة النقدية اليومية والشهرية بما في ذلك فواتير الكاش، المقدمات، وتحصيلات الأقساط. | `/reports/cash-drawer/` |
| 3 | **المصروفات (Expenses)** | سرد مفصل لجميع المصروفات التشغيلية خلال فترة معينة مع إمكانية التحديد والطباعة. | `/expenses/` |
| 4 | **تحصيلات الأقساط (Installments)** | عرض تفصيلي لعمليات تحصيل الأقساط مع إمكانيات فرز وبحث متقدمة (بالمندوب، المنطقة، العميل، الخ) وإجمالي التحصيلات. | `/reports/installments/` |
| 5 | **جرد المخزن (Inventory Movement)** | مراقبة حركة الأصناف في المخزن (الكمية الافتتاحية، المشتريات، المرتجعات، المبيعات، الزيادة والعجز) وحساب قيمة المخزون الحالية. | `/reports/inventory-movement/` |
| 6 | **الأرباح والخسائر (Profit and Loss)** | تحليل ربحية كل صنف مباع على حدة متضمناً سعر البيع، التكلفة، عمولات البيع والتحصيل، وصافي المكسب. | `/reports/profit-and-loss/` |
| 7 | **الفواتير / المبيعات (Receipts)** | قائمة بجميع الفواتير (كاش أو قسط أو بها مقدمات) مع إمكانية البحث المفصل حسب البائع، العميل، أو أرقام الفواتير. | `/receipts/` |
| 8 | **أداء المناديب (Salesperson Performance)** | تقييم شامل لأداء كل مندوب يشمل مبيعات الكاش والقسط، إجمالي التحصيل، العمولات المستحقة والمرتب النهائي المطلوب سداده. | `/reports/salesperson-performance/` |

---

## تفاصيل التقارير

### 1. ملخص لوحة القيادة (Dashboard Report)

- **المكوّن**: `DashboardReport.jsx`
- **الغرض**: يعرض ملخصاً شاملاً لأداء الفرع خلال الشهر المحدد
- **البيانات المعروضة**:
  - إجمالي المبيعات (كاش + أقساط)
  - تكلفة البضاعة المباعة
  - إجمالي المصروفات التشغيلية
  - صافي المكسب
  - تفاصيل حركة السيولة النقدية

### 2. درج الخزينة (Cash Drawer Report)

- **المكوّن**: `CashDrawerReport.jsx`
- **الغرض**: تتبع حركة النقدية الفعلية في الخزينة
- **البيانات المعروضة**:
  - فواتير الكاش المحصّلة
  - مقدمات الأقساط المحصّلة
  - تحصيلات أقساط الشهر
  - إجمالي الوارد للخزينة

### 3. تقرير المصروفات (Expenses Report)

- **المكوّن**: `ExpensesReport.jsx`
- **الغرض**: سرد وتصنيف جميع المصروفات التشغيلية
- **إمكانيات**: تصفية بالفترة، طباعة مباشرة

### 4. تقرير تحصيلات الأقساط (Installments Report)

- **المكوّن**: `InstallmentsReport.jsx`
- **الغرض**: عرض تفصيلي لعمليات التحصيل
- **إمكانيات البحث**: بالمندوب، المنطقة، العميل، رقم الفاتورة
- **البيانات**: إجمالي التحصيلات، تفاصيل كل عملية تحصيل

### 5. تقرير جرد المخزن (Inventory Movement Report)

- **المكوّن**: `InventoryMovementReport.jsx`
- **الغرض**: مراقبة حركة كل صنف في المخزن
- **الأعمدة**:
  - الكمية الافتتاحية
  - المشتريات (الوارد)
  - المرتجعات
  - المبيعات (المنصرف)
  - الزيادة والعجز (التسوية)
  - الرصيد الحالي
  - قيمة المخزون

### 6. تقرير الأرباح والخسائر (Profit and Loss Report)

- **المكوّن**: `ProfitAndLossReport.jsx`
- **الغرض**: تحليل ربحية كل صنف مباع
- **الأعمدة**:
  - سعر البيع
  - سعر التكلفة
  - عمولة البيع (مندبة)
  - عمولة التحصيل
  - صافي المكسب لكل صنف

### 7. تقرير الفواتير / المبيعات (Receipts Report)

- **المكوّن**: `ReceiptsReport.jsx`
- **الغرض**: عرض جميع الفواتير المسجلة
- **إمكانيات البحث**: نوع الفاتورة (كاش/قسط)، البائع، العميل، رقم الفاتورة
- **البيانات**: تاريخ الفاتورة، المبلغ، نوع الدفع، المندوب

### 8. تقرير أداء المناديب (Salesperson Performance Report)

- **المكوّن**: `SalespersonPerformanceReport.jsx`
- **الغرض**: تقييم شامل لأداء المندوب وحساب المرتب المستحق
- **مسار الـ API**: `/reports/salesperson-performance/`

## حساب عمولات المندوب (Commission Calculation)

تقرير أداء المناديب يعتمد على نوعين مختلفين من العمولات:

### أ. عمولة البيع — مندبة (Sales Commission)

- **النوع**: مبلغ ثابت لكل صنف
- **الحقل**: `commission` على نموذج `ReceiptItem` (يُنسخ من `InventoryItem.sales_commission` وقت البيع)
- **الحساب**:

```
عمولة البيع لكل صنف = sales_commission × quantity
إجمالي عمولة البيع = مجموع (sales_commission × quantity) لجميع الأصناف المباعة
```

- **مثال**: صنف عمولته ٥ جنيه، بيع منه ١٠ وحدات → عمولة البيع = ٥٠ جنيه

### ب. عمولة التحصيل — عمولة تحصيل (Collection Commission)

- **النوع**: نسبة مئوية على مستوى الفرع
- **الحقل**: `collection_commission_rate` على نموذج `Branch`
- **الحساب**:

```
عمولة التحصيل = إجمالي التحصيلات الشهرية للمندوب × (collection_commission_rate / 100)
```

- **مثال**: مندوب حصّل ١٠,٠٠٠ جنيه، نسبة عمولة الفرع ٢٪ → عمولة التحصيل = ٢٠٠ جنيه

### ج. المرتب النهائي (Total Salary)

> [!IMPORTANT]
> المندوب في VentaPOS يعمل بنظام العمولة فقط — لا يوجد مرتب أساسي.

```
المرتب النهائي = إجمالي عمولة البيع (مندبة) + عمولة التحصيل
```

### البيانات المعروضة في التقرير

| العمود | الوصف |
| :--- | :--- |
| اسم المندوب | اسم المندوب |
| مبيعات الكاش | إجمالي مبيعات الكاش للشهر |
| مبيعات القسط | إجمالي مبيعات الأقساط للشهر |
| إجمالي التحصيل | المبالغ المحصّلة من أقساط سابقة |
| عمولة البيع (مندبة) | مبلغ ثابت × عدد الوحدات المباعة |
| عمولة التحصيل | نسبة الفرع × إجمالي التحصيل |
| المرتب المستحق | مجموع العمولتين |

## ملاحظات فنية

- جميع التقارير تعمل بنظام **الشهر/السنة** (Monthly-based) وليس يومياً
- التقارير مرتبطة بالفرع المحدد (Branch-scoped)
- الطباعة تتم عبر `os.startfile()` / SumatraPDF (طباعة محلية صامتة)
- **Navigation**: `ReportsIndex.jsx` يوفر فهرس التقارير، و`ReportsLayout.jsx` يوفر الإطار العام
