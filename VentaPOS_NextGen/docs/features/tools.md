# أدوات النظام (System Tools)

يوفر نظام VentaPOS مجموعة من الأدوات الحيوية لإدارة المزامنة مع الأجهزة المحمولة، النسخ الاحتياطي، واستيراد البيانات.

- **المكوّن الرئيسي**: `ToolsDashboard.jsx`
- **المسار**: `/tools/*`

---

## 1. أداة المزامنة (Sync Tool)

تُتيح أداة المزامنة (`SyncTab`) ربط النظام المركزي بالأجهزة المحمولة للمناديب، وتوفر طريقتين للتعامل بناءً على توفر الإنترنت وحالة الاشتراك.

### أ. الموبايل المحلّي (Offline Sync)

تُستخدم هذه الطريقة للأجهزة التي تعمل **بدون اتصال بالإنترنت**. يتم نقل البيانات عبر USB أو تنزيل الملفات يدوياً.

#### تصدير ملف تهيئة الموبايل

| البند | القيمة |
| :--- | :--- |
| **الغرض** | تنزيل ملف إعدادات (JSON) يحتوي على الأصناف وبيانات المندوب لتعمل محلياً على هاتفه |
| **الإجراء** | اختيار المندوب → تنزيل الملف → نقله للهاتف عبر USB |
| **المسار** | `GET /api/v1/tools/sync/export-items/` |

#### استيراد فواتير الموبايل

| البند | القيمة |
| :--- | :--- |
| **الغرض** | سحب الفواتير من هاتف المندوب كملف JSON ورفعه للنظام المركزي لدمج المبيعات |
| **الإجراء** | نسخ ملف الفواتير من الهاتف → رفعه للنظام |
| **المسار** | `POST /api/v1/tools/sync/import-receipts/` |

### ب. الربط السحابي (Online Sync)

> [!WARNING]
> هذه الميزة تتطلب **تكامل سحابي مستقبلي** (Cloud Sync Server) وهو حالياً **مؤجل** وغير مُنفّذ. الوصف التالي يوثّق التصميم المخطط له.

تتطلب هذه الميزة اشتراكاً فعالاً في الباقة السحابية (Online / Pro). تقوم الأداة أولاً بالتحقق من حالة الترخيص عبر `/api/v1/license/status/`.

#### إدارة الأجهزة المساعدة

- يمكن تفعيل أو إيقاف وصول أجهزة المناديب للنظام المركزي في الوقت الفعلي
- **المسار**: `/api/v1/salespersons/`

#### الفواتير المعلقة (Pending Receipts)

- يستقبل النظام الفواتير من الأجهزة فورياً وتُعرض كـ "فواتير بانتظار الاعتماد"
- يمكن للمدير مراجعتها ثم الضغط على "اعتماد" لإدخالها رسمياً في الحسابات
- **المسار**: `POST /api/v1/tools/sync/approve-pending/<int:pk>/`

## 2. أداة النسخ الاحتياطي (Backup Tool)

صُممت أداة النسخ الاحتياطي (`BackupTab`) لضمان أمان البيانات واستمراريتها. تتعامل مباشرة مع قاعدة بيانات SQLite المدمجة.

### تنزيل نسخة احتياطية (Hot Backup)

| البند | القيمة |
| :--- | :--- |
| **الغرض** | أخذ نسخة احتياطية فورية من قاعدة البيانات بدون إيقاف النظام |
| **الصيغة** | ملف `.sqlite3` |
| **المسار** | `GET /api/v1/tools/backup/download/` |
| **ملاحظة** | يستخدم SQLite Hot Backup API لضمان تناسق البيانات أثناء النسخ |

### استعادة النظام (Restore Backup)

| البند | القيمة |
| :--- | :--- |
| **الغرض** | رفع ملف نسخة احتياطية سابق لاستعادة البيانات |
| **الصيغ المقبولة** | `.sqlite3` أو `.db` |
| **المسار** | `POST /api/v1/tools/backup/upload/` |

**خطوات الاستعادة:**

1. رفع ملف النسخة الاحتياطية
2. **فحص سلامة (Integrity Check)** — يتحقق النظام من صحة الملف وسلامة بنيته
3. **جدولة التطبيق** — لا يتم الكتابة فوق البيانات مباشرة، بل يُجدوَل التطبيق عند إعادة التشغيل القادمة
4. **إعادة التشغيل** — عند التشغيل التالي يتم تطبيق النسخة الاحتياطية بأمان

> [!CAUTION]
> عملية الاستعادة تستبدل **جميع** البيانات الحالية بالبيانات الموجودة في النسخة الاحتياطية. يُنصح بأخذ نسخة احتياطية من البيانات الحالية قبل الاستعادة.

## 3. أداة الاستيراد الذكي (Smart Import)

تتيح أداة الاستيراد الذكي استيراد الأصناف بشكل مجمّع من ملفات Excel، وهي مفيدة بشكل خاص عند بداية استخدام النظام.

### معلومات الأداة

| البند | القيمة |
| :--- | :--- |
| **المكوّن** | `SmartImportTab.jsx` |
| **المسار** | `POST /api/v1/tools/smart-import/` |
| **المعالج** | `SmartImportWizardView` في `api/tools_views.py` |
| **المكتبة** | `pandas` (`pd.read_excel`) |
| **الصيغ المقبولة** | `.xlsx` أو `.xls` |

### أعمدة الملف المطلوبة

| عمود Excel (عربي) | الحقل في النظام | مطلوب؟ | القيمة الافتراضية |
| :--- | :--- | :--- | :--- |
| **اسم الصنف** | `name` | ✅ نعم | (يتم تخطي الصف إذا كان فارغاً) |
| **الكمية** | `available_quantity` | لا | `0` |

> [!IMPORTANT]
> الاستيراد الذكي يستورد **الاسم والكمية فقط**. لا يوجد حقل سعر بيع ثابت على المنتج — الكاشير يُدخل السعر وقت البيع.

### المعالجة في الخلفية

1. قراءة ملف Excel باستخدام `pandas`
2. التكرار على الصفوف داخل `transaction.atomic()`
3. استخدام `update_or_create` على نموذج `InventoryItem`:
   - إذا وُجد صنف بنفس الاسم في الفرع الحالي → يتم تحديث الكمية
   - إذا لم يُوجد → يتم إنشاء سجل جديد
